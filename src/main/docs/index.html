Overview

Dev Shell is a powerful command-line interface that enhances your Git workflow with intelligent features and a superior user experience. Built with Spring Shell and JGit, it provides a safer, more intuitive way to manage your Git repositories.

Safe Commits

Automatic branch management prevents accidental commits to the wrong branch

Smart Workflow

Intelligent commit process with file tracking and automatic staging

Better UX

Emoji-enhanced output and clear status indicators for better visibility

Installation

Prerequisites

  • Java 24 or higher
  • Maven 3.8+
  • Git installed on your system

Build and Run

# Clone the repository
git clone https://github.com/yourusername/dev-shell.git
cd dev-shell

# Build the project
./mvnw clean package

# Run the application
./mvnw spring-boot:run

Create Executable JAR

# Build executable JAR
./mvnw clean package

# Run the JAR
java -jar target/dev-shell-0.0.1-SNAPSHOT.jar

Available Commands

Navigation Commands

pwd

Print current working directory

cd [path]

Change directory (supports ~, .., relative and absolute paths)

ls [path]

List directory contents with icons for files and folders

Git Commands

commit "message" [--push]

Execute smart commit workflow with optional push to remote

shell:> commit "Add new feature" --push
status

Show repository status with staged, unstaged, and untracked files

add [--all] [--files "file1 file2"]

Add untracked files to staging area

shell:> add --all
shell:> add --files "src/Main.java README.md"
log [--count N]

Show recent commit history (default: 10 commits)

git-init [name]

Initialize a new Git repository

Configuration Commands

config --name "Name" --email "email"

Configure Git user settings

auth

Interactive authentication setup

validate

Validate repository configuration and readiness

git-help

Display comprehensive help information

Smart Commit Workflow

The Smart Commit feature is the flagship capability of Dev Shell. It implements a safe, automated workflow that prevents common Git mistakes and ensures clean commit history.

How It Works

1

Creates Temporary Branch

Automatically creates a temporary branch to isolate your changes

2

Stages All Files

Automatically stages all tracked files (optionally includes untracked files)

3

Enhanced Commit Message

Appends a complete list of changed files to your commit message

4

Safe Merge

Merges the temporary branch back to your original branch

5

Cleanup

Deletes the temporary branch and optionally pushes to remote

- README.md

Architecture

Dev Shell follows Domain-Driven Design (DDD) principles with a clean, layered architecture that promotes maintainability and testability.

Domain Layer

  • Models: Repository, Branch, Commit, WorkingDirectory
  • Value Objects: Author, BranchName, CommitMessage
  • Services: SmartCommitService, GitValidationService
  • Repository Interface: GitRepository
  • Exceptions: Custom domain exceptions with factory pattern

Infrastructure Layer

  • JGitRepositoryAdapter: Implements GitRepository using JGit
  • External Libraries: JGit for Git operations

Presentation Layer

  • GitCommands: Spring Shell component with all CLI commands
  • User Interface: Terminal-based with emoji enhancements

Technology Stack

  • Spring Boot 3.4.7 - Application framework
  • Spring Shell 3.4.0 - CLI framework
  • JGit 7.3.0 - Java Git implementation
  • Java 24 - Programming language
  • Maven - Build tool

Application Properties

# Logging Configuration

logging.level.root=INFO
logging.level.io.joshuasalcedo.homelab.devshell=DEBUG

spring.shell.interactive.enabled=true
spring.main.banner-mode=off
spring.main.web-application-type=none

Error Codes

Dev Shell provides descriptive error messages with specific error codes for easier troubleshooting:

REPO_NOT_FOUND Repository not found at the specified path
INVALID_BRANCH Invalid branch name or branch operation failed
COMMIT_FAILED Commit operation failed
AUTH_REQUIRED Git authentication not configured
VALIDATION_ERROR Repository validation failed
